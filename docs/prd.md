# PRD: askdocs

> **Правило роботи зі статусами.** Агент реалізує ТІЛЬКИ вимоги зі статусом
> `accepted`. Вимоги зі статусом `proposed` — записані для повноти мапи, але
> НЕ реалізуються, поки власник продукту вручну не перемкне статус.
> Значення Status: `proposed` / `accepted` / `shipped` / `dropped`.

## Продукт

**askdocs** — локальний RAG-інструмент "чат з документацією". Вхід — корпус
локальних `.md` файлів. Користувач ставить питання в CLI й отримує відповідь,
що спирається ТІЛЬКИ на корпус, із посиланням на файл-джерело. Якщо відповіді
в корпусі немає — система чесно каже про це, а не вигадує.

## Домен тестового корпусу

**ГущоЛіт** — вигаданий проєкт літаючих авто на кавовій гущі. Домен навмисно
синтетичний: правильна відповідь може прийти лише з корпусу, не зі знань
моделі. Якщо система відповідає з "загальних знань" — це помилка.

## Глосарій

- **Chunk** — фрагмент документа, одиниця зберігання й пошуку у vector store.
- **Golden set** — фіксований набір питань із відомими правильними відповідями по корпусу, база для eval.
- **DocSource / Retriever / LLMProvider** — три архітектурні інтерфейси: "дай документи" / "знайди релевантне під питання" / "згенеруй відповідь з контексту".
