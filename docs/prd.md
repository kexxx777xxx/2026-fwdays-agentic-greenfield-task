# PRD: askdocs

> **Правило роботи зі статусами.** Агент реалізує ТІЛЬКИ вимоги зі статусом
> `accepted`. Вимоги зі статусом `proposed` — записані для повноти мапи, але
> НЕ реалізуються, поки власник продукту вручну не перемкне статус.
> Значення Status: `proposed` / `accepted` / `shipped` / `dropped`.

## Продукт

**askdocs** — локальний RAG-інструмент "чат з документацією". Вхід — корпус
локальних `.md` файлів. Користувач ставить питання в CLI й отримує відповідь,
що спирається ТІЛЬКИ на корпус, із посиланням на файл-джерело. Якщо відповіді
в корпусі немає — система чесно каже про це, а не вигадує.

## Домен тестового корпусу

**ГущоЛіт** — вигаданий проєкт літаючих авто на кавовій гущі. Домен навмисно
синтетичний: правильна відповідь може прийти лише з корпусу, не зі знань
моделі. Якщо система відповідає з "загальних знань" — це помилка.

## Нефункціональні вимоги (NFR)

| ID | Вимога | Status |
|---|---|---|
| NFR-001 | Відповідь без посилання на джерело = баг. | accepted |
| NFR-002 | "Немає в корпусі" — валідна відповідь, не помилка. | accepted |
| NFR-003 | Критерій готовності capability — зелені тести (pytest), не "здається працює". | accepted |
| NFR-004 | Система не відповідає із "загальних знань" моделі поза корпусом; синтетичний домен ГущоЛіт існує саме для перевірки цього. | accepted |
| NFR-006 | Збій зовнішніх залежностей (LLM, vector store) не роняє користувацькі поверхні трейсбеком: CLI показує зрозуміле повідомлення й не рве REPL; sync-watcher переживає транзієнтний збій проходу і некоректну конфігурацію інтервалу. | shipped |
| NFR-007 | Помилка інфраструктури не видається за валідну відмову «немає в корпусі»: збій LLM повертається як окремий контрольований error-стан (прапорець `error`), відмінний від відмови `found=False`, яка лишається твердженням саме про відсутність у корпусі. | shipped |
| NFR-008 | Контейнер застосунку працює від non-root користувача (монтує репо та корпус, тож RCE не має отримувати root-доступ до host-файлів); назва embedding-моделі має єдине джерело істини (`askdocs/embeddings.py`), а не дублюється в образі. | shipped |

## Глосарій

- **Chunk** — фрагмент документа, одиниця зберігання й пошуку у vector store.
- **Golden set** — фіксований набір питань із відомими правильними відповідями по корпусу, база для eval.
- **DocSource / Retriever / LLMProvider** — три архітектурні інтерфейси: "дай документи" / "знайди релевантне під питання" / "згенеруй відповідь з контексту".
